Installation

Requirements

  • Python 3.10 or higher
  • PyTorch 2.0 or higher
  • CUDA-capable GPU (recommended, but CPU works)

pip install (Recommended)

pip install frustrampnn[viz]

From source

git clone https://github.com/schoederlab/frustraMPNN.git
cd frustraMPNN
pip install -e ".[all]"

Using uv (fast)

curl -LsSf https://astral.sh/uv/install.sh | sh
git clone https://github.com/schoederlab/frustraMPNN.git
cd frustraMPNN
uv sync --all-extras

Quick Start

Basic Usage

from frustrampnn import FrustraMPNN

# Load model
model = FrustraMPNN.from_pretrained("checkpoint.ckpt")

# Predict frustration for all positions
results = model.predict("protein.pdb")

# View results
print(results.head(20))

Output Format

Results are returned as a pandas DataFrame:

   frustration_pred  position wildtype mutation   pdb chain
0          0.334012         0        M        A  1UBQ     A
1          1.410234         0        M        C  1UBQ     A
2         -0.892341         0        M        D  1UBQ     A
...
Column Description
frustration_pred Predicted frustration index
position 0-indexed residue position
wildtype Original amino acid (1-letter code)
mutation Substituted amino acid
pdb PDB identifier
chain Chain identifier

Frustration Categories

Category Frustration Index Interpretation
Highly frustrated ≤ −1.0 Conflicting interactions, often functional
Neutral −1.0 to 0.58 Average interactions
Minimally frustrated ≥ 0.58 Optimized interactions, stable

Filter Native Residues

# Get native residue frustration (wildtype == mutation)
native = results[results['wildtype'] == results['mutation']]

# Find highly frustrated positions
highly_frustrated = native[native['frustration_pred'] <= -1.0]
print(f"Highly frustrated positions: {highly_frustrated['position'].tolist()}")

Python API

FrustraMPNN Class

from frustrampnn import FrustraMPNN

# Load from checkpoint
model = FrustraMPNN.from_pretrained("checkpoint.ckpt")

# Load with specific device
model = FrustraMPNN.from_pretrained("checkpoint.ckpt", device="cuda")
model = FrustraMPNN.from_pretrained("checkpoint.ckpt", device="cpu")

predict()

results = model.predict(
    pdb_path="protein.pdb",
    chains=["A"],           # Optional: specific chains
    show_progress=True,     # Show progress bar
)

predict_batch()

from pathlib import Path

pdb_files = list(Path("structures/").glob("*.pdb"))
results = model.predict_batch(
    [str(f) for f in pdb_files],
    chains=["A"],
    show_progress=True
)
results.to_csv("all_results.csv", index=False)

Batch Processing

from frustrampnn import FrustraMPNN
from pathlib import Path
import pandas as pd

model = FrustraMPNN.from_pretrained("checkpoint.ckpt")

all_results = []
for pdb_file in Path("structures/").glob("*.pdb"):
    try:
        results = model.predict(str(pdb_file), chains=["A"])
        all_results.append(results)
    except Exception as e:
        print(f"Failed {pdb_file.name}: {e}")

combined = pd.concat(all_results, ignore_index=True)
combined.to_csv("all_results.csv", index=False)

Command Line Interface

Basic Prediction

frustrampnn predict \
    --pdb protein.pdb \
    --checkpoint model.ckpt \
    --output results.csv

Batch Processing

frustrampnn batch ./structures/ \
    --checkpoint model.ckpt \
    --output-dir ./results/ \
    --pattern "*.pdb" \
    --chains A \
    --device cuda

Options

Option Description
--pdb Path to PDB file
--checkpoint Path to model checkpoint
--output Output file path
--chains Chain(s) to analyze
--device Device (cuda, cpu)
--format Output format (csv, json)

Visualization

Single-Residue Plot

Show frustration for all 20 amino acids at a specific position:

from frustrampnn.visualization import plot_single_residue

fig = plot_single_residue(
    results,
    position=72,
    chain="A",
    title="Position 73 - Active Site"
)
fig.savefig("position_73.png", dpi=300, bbox_inches='tight')

Interactive Plot (Plotly)

FrustraMPNN uses a consistent color scheme matching frustrapy:

Highly frustrated (≤ −1.0)
Neutral (−1.0 to 0.58)
Minimally frustrated (≥ 0.58)

Validation

Compare FrustraMPNN predictions with physics-based FrustratometeR calculations:

from frustrampnn import FrustraMPNN
from frustrampnn.validation import compare_with_frustrapy

# Load model and predict
model = FrustraMPNN.from_pretrained("checkpoint.ckpt")
results = model.predict("protein.pdb", chains=["A"])

# Compare with frustrapy (slow - calculates physics-based frustration)
comparison = compare_with_frustrapy(
    pdb_path="protein.pdb",
    chain="A",
    frustrampnn_results=results,
)

# View metrics
print(f"Spearman correlation: {comparison.spearman:.3f}")
print(f"Pearson correlation: {comparison.pearson:.3f}")
print(f"RMSE: {comparison.rmse:.3f}")

Specific Positions

For large proteins, compare only specific positions:

comparison = compare_with_frustrapy(
    pdb_path="protein.pdb",
    chain="A",
    frustrampnn_results=results,
    positions=[50, 73, 120, 200],  # Only these positions
)

Visualization

fig = comparison.plot(
    show_regression=True,
    show_identity=True,
    title="FrustraMPNN vs FrustratometeR"
)
fig.savefig("comparison.png", dpi=300)

Docker and Singularity

Docker

# Build image
cd docker
bash build_docker.sh

# Run prediction
bash docker/run_docker.sh predict \
    --pdb /data/protein.pdb \
    --checkpoint /data/model.ckpt \
    --output /data/results.csv

With GPU

docker run --gpus all -v $(pwd):/data frustrampnn:latest \
    frustrampnn predict \
    --pdb /data/protein.pdb \
    --checkpoint /data/model.ckpt

Singularity (HPC)

# Build from Docker
bash docker/build_singu_from_docker.sh

# Run on cluster
singularity exec --nv frustrampnn.sif frustrampnn predict \
    --pdb protein.pdb \
    --checkpoint model.ckpt

SLURM Job Script

#!/bin/bash
#SBATCH --job-name=frustrampnn
#SBATCH --partition=gpu
#SBATCH --gres=gpu:1
#SBATCH --cpus-per-task=4
#SBATCH --mem=16G
#SBATCH --time=4:00:00

module load singularity

singularity exec --nv \
    --bind $PWD:/data \
    /path/to/frustrampnn.sif \
    frustrampnn predict \
    --pdb /data/protein.pdb \
    --checkpoint /data/model.ckpt \
    --output /data/results.csv
